## 1. 系统概述

### 1.1 核心定位
财神爷量化交易系统是一套基于多Agent协作的自动化量化交易平台，通过8个独立Agent的分工协作，实现从数据采集、舆情分析、量化打分、初筛选股、预警推送、模拟交易到策略进化的完整闭环。

### 1.2 系统目标
- **核心KPI**: 月收益率 > 5%
- **风险控制**: 最大回撤 < 10%
- **胜率目标**: > 55%
- **进化能力**: 基于实际交易结果持续优化策略

### 2.3 南风 (NanFeng) - 量化大脑 Agent

**职责**: 技术指标计算、量化打分、信号生成

**核心KPI**: 月收益率 > 5%

**技术指标体系**:

| 指标 | 权重 | 说明 |
|------|------|------|
| 均线系统 | 30% | MA5/MA10/MA20多头排列 |
| MACD | 20% | DIF/DEA金叉、零轴上方 |
| RSI | 15% | 50-70区间（强势） |
| 成交量 | 15% | 温和放量1.2-2.5倍 |
| ATR | 10% | 波动率1%-5% |
| 形态 | 10% | 突破、支撑/阻力 |

**打分公式**:
```python
def calculate_score(stock_data):
    score = 0
    
    # 1. 均线系统 (30分)
    if price > ma5 > ma10:
        score += 20
    if price > ma20:
        score += 10
    
    # 2. MACD (20分)
    if macd > 0:
        score += 10
    if dif > dea > 0:
        score += 10
    
    # 3. RSI (15分)
    if 50 < rsi < 70:
        score += 15
    
    # 4. 成交量 (15分)
    if 1.2 <= volume_ratio <= 2.5:
        score += 15
    
    # 5. ATR (10分)
    if 0.01 <= atr_pct <= 0.05:
        score += 10
    
    # 6. 题材加分 (额外)
    if is_hot_sector:
        score += 5
    
    return min(score, 10)  # 满分10分
```

    "score": 8.5,

### 2.4 东风 (DongFeng) - 初筛选股 Agent

**职责**: 交易时段实时监控、初筛活跃股票、管理候选池

**监控时段**: 13:30-14:15（收盘前45-90分钟）

**筛选条件**:

| 条件 | 阈值 | 说明 |
|------|------|------|
| 振幅 | > 3% | (最高-最低)/昨收 |
| 温和放量 | 1.2-2.5倍 | 当日成交量/5日均量 |
| 题材对齐 | High/Medium | 西风热点板块优先 |

**双池管理**:

1. **候选池** (`candidate_pool.json`)
   - 进入条件：满足全部筛选条件
   - 记录：代码、名称、进入时间、振幅、放量倍数
   - 用途：供红中/南风进一步筛选

2. **备用池** (`standby_pool.json`)
   - 进入条件：曾进入候选池但不再活跃
   - 记录：代码、退出时间、退出原因、停留时长
   - 原因：振幅收窄、量能枯竭、热点漂移

**轮询频率**: 每15分钟扫描一次

**输出**: `candidate_pool.json` → 供红中/南风使用

### 2.6 发财 (FaCai) - 模拟交易 Agent

**职责**: 模拟交易执行、动态风控、资产管理

**账户设置**:
- 初始资金: 100,000 CNY
- 单只股票持仓上限: 50%
- 最小买入金额: 1,000 CNY
- 交易单位: 100股整数

**买入规则**:
- **时间**: 14:50-15:00（收盘前10分钟）
- **条件**: 南风打分 >= 8.5 且红中已预警
- **禁止**: 集合竞价时间（9:15-9:25）不能交易

**卖出规则**:
- **手续费**: 万分之三（0.03%）
- **净盈亏**: 已扣除手续费
- **禁止**: 集合竞价时间不能交易

**动态风控**:

1. **追踪止损** (Trailing Stop Loss)
   - 初始止损: 买入价 -5%
   - 随盈利上移: 止损位 = 买入价 + (最高价 - 买入价) × 0.5
   - 目的: 锁定既得利益

2. **情绪止盈**
   - 触发条件: 西风热点评分从"High"降至"Low"
   - 动作: 强制减仓或清仓
   - 目的: 规避情绪退潮风险

3. **波动止损**
   - 触发条件: 当日振幅 > 8% 且向下突破
   - 动作: 提前清仓
   - 目的: 规避异常波动风险

**账户记录**:
```sql
CREATE TABLE trades (
    id INTEGER PRIMARY KEY,
    timestamp TIMESTAMP,
    action TEXT,  -- BUY/SELL
    symbol TEXT,
    name TEXT,
    price REAL,
    quantity INTEGER,
    total_amount REAL,
    fee REAL,  -- 卖出手续费
    logic TEXT,
    total_assets REAL,
    cash_balance REAL
);
```

**运行时间**: 14:50-15:00（买入），持续监控（风控）

### 2.7 白板 (BaiBan) - 策略进化 Agent

**职责**: 收盘后/周末全量回测、归因分析、策略进化

**核心功能**:

1. **归因分析** (Attribution Analysis)
   - 胜率拆解: 西风题材 vs 南风技术
   - 失败复盘: 止损原因分析
   - 收益归因: 各因子贡献度

2. **市场情绪感知** (Market Sentiment Sensor)

| 情绪 | 判定标准 | 策略调整 |
|------|----------|----------|
| **亢奋 (Bull)** | MA20向上，上涨>3000家 | 题材权重+20%，止盈15% |
| **震荡 (Flat)** | BOLL中轨，成交持平 | 严格15分钟买入，止盈5% |
| **恐慌 (Bear)** | 跌破MA60，跌停激增 | 初始分-2，止损3%，仓位20% |

3. **参数优化**
   - 动态止盈步长优化
   - 打分权重自适应调整
   - 风控参数动态校准

4. **进化指令生成**

每周生成《策略进化清单》:
```
📈 [白板·策略进化指令]
● 本周回测表现: 胜率58%，回撤2.1%，资产增值4.5%
● 情绪诊断: 当前处于【震荡转向恐慌】阶段
● 进化调整:
  - 南风: 将"大盘破位"权重提高1.5倍
  - 东风: 活跃度门槛从3%提高到4%
  - 发财: 强制降低单笔仓位至20%
[下次全量回测时间: 周日 20:00]
```

**执行时间**:
- 每日收盘后 (15:30): 归因分析+情绪诊断
- 每周日 20:00: 全量回测+生成进化指令

## 5. KPI 与进化

### 5.1 核心KPI

| KPI | 目标 | 监控频率 |
|-----|------|----------|
| 月收益率 | > 5% | 每日 |
| 最大回撤 | < 10% | 每日 |
| 胜率 | > 55% | 每周 |
| 夏普比率 | > 1.5 | 每月 |

### 5.2 进化机制

**未达标时自动优化**:

1. **南风权重调整**
   - 技术分表现差 → 降低Tech权重
   - 题材分表现差 → 降低Theme权重
   - 根据白板归因结果动态调整

2. **东风阈值调整**
   - 入场过早被套 → 提高振幅阈值
   - 错过机会 → 降低振幅阈值
   - 量能判断不准 → 调整放量区间

3. **发财风控调整**
   - 止损过于频繁 → 放宽止损位
   - 回撤过大 → 收紧止损位
   - 情绪判断失误 → 调整热点跟踪敏感度
