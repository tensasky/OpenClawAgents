## 📋 版本管理规范

### Git标签命名
```
{agent}-v{主版本}.{次版本}.{修订}
例如: beifeng-v3.0.0, nanfeng-v5.1.0
```

### 版本状态
- ✅ 运行中 - 正在工作
- ⏸️ 待机 - 等待触发
- 🔧 维护中 - 修复问题
- ❌ 故障 - 需要修复

### 更新流程
1. 开发测试 → 2. 提交代码 → 3. 创建标签 → 4. 更新文档 → 5. 推送GitHub
