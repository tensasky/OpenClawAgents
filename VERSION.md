# 版本管理说明

## 版本号规则
采用语义化版本控制 (Semantic Versioning)

格式：`MAJOR.MINOR.PATCH`

- **MAJOR** (主版本): 重大功能变更，不兼容的API修改
- **MINOR** (次版本): 新增功能，向后兼容
- **PATCH** (补丁): Bug修复，向后兼容

```bash
# 创建新版本
git tag -a v4.2.0 -m "描述"
```
